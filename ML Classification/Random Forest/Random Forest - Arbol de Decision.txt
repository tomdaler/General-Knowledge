Proceso de decision: como los modelos hacen una prediccion o retornan una respuesta, usando parametros

Funcion error/coste: como evaluar si los parametros en el modelo generan buenas predicciones

Regla de actualizacion: como mejorar los parametros para hacer mejores predicciones (usando optimizacion numerica)

Del DataSet Original
====================

Validation (0-20) verificar modelo aprende

Random Forest: Decision tree

PARAMETROS
- Mas arboles menor la variacion, pero mas computo
- Features: numero propiedades para partir ramas
- Profundidad: niveles

- N-split: (para dividirse) Min sample split: No. de data points que nodo debe tener antes de dividirse
- n-min: (para crear hoja) Min samples leaf: Minimo No. de muestras requeridas para estar en una hoja (leaf)

Funcion de costo: para un feature seleccionado encuentra el menor split-point (punto de separacion) y separa los datos en 2 nodos mas pequenos

Regla de actualizacion: si el nodo hoja tiene mas de n-min puntos de datos, repite fx de coste, sino se detiene.
